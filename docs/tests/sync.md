# `spectrack sync` コマンド テストケース一覧

## 1. 基本動作（正常系：バージョンとパスの同期）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **S1-1** | バージョンの同期（単一） | 対象ファイルが `b.md:1.0.0` に依存。`b.md` の Working tree 上のバージョンは `1.1.0` | `spectrack sync docs/a.md` | 対象ファイルの依存リストにある `b.md` の参照バージョンが `1.1.0` に上書き更新されること |
| **S1-2** | バージョンの同期（複数） | 対象ファイルが `b.md:1.0`, `c.md:2.0` に依存。Working tree では `b.md:1.1`, `c.md:2.1` に更新済み | `spectrack sync docs/a.md` | `b.md`, `c.md` 両方の参照バージョンが一括で最新の Working tree バージョンに同期されること |
| **S1-3** | ファイル移動（リネーム）への追従とパスヒント更新 | 依存先 `b.md` が `docs/domain/b-renamed.md` に移動（リネーム）されている（IDは同一） | `spectrack sync docs/a.md` | IDベースで移動先ファイルが正しく発見され、対象ファイルの依存リスト内の `path` ヒントが新しいパスに自動更新されること |
| **S1-4** | AST（フォーマット・コメント）の維持 | 依存リストの横にYAMLコメントがある | `spectrack sync docs/a.md` | `version` や `path` の値のみが書き換わり、コメントや独自のインデント形式がそのまま維持されること |

## 2. `--only` オプションの動作（Filtering Options）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **O2-1** | パス指定での個別同期 | 対象ファイルが `b.md`, `c.md` に依存。両方とも更新されている | `spectrack sync docs/a.md --only=docs/b.md` | `b.md` の参照バージョンのみが同期され、`c.md` は古いバージョンのまま維持されること |
| **O2-2** | ID指定での個別同期 | 対象ファイルが `b.md` (ID: st-b-001) に依存。更新されている | `spectrack sync docs/a.md --only=st-b-001` | ID指定で正しく依存先がフィルタリングされ、`b.md` のみが同期されること |
| **O2-3** | 複数指定での個別同期 | 依存先 `b.md`, `c.md`, `d.md` がすべて更新済み | `spectrack sync docs/a.md --only=docs/b.md,st-c-002` | カンマ区切りでパスとIDを混在指定しても正しく解釈され、指定した2ファイルのみが同期されること |
| **O2-4** | 存在しない・依存していないファイルの指定 | 対象ファイルは `b.md` に依存しているが、`--only=docs/x.md` を指定 | `spectrack sync docs/a.md --only=docs/x.md` | 「指定された依存関係はありません」等の警告メッセージが表示され、ファイルが変更されず正常終了すること |

## 3. 冪等性と状態変化（Idempotency / State Changes）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **I3-1** | 変更がない場合の同期 | 対象ファイルの依存先バージョン・パスが、Working tree と完全に一致している | `spectrack sync docs/a.md` | 「すべて最新です」等のメッセージが表示され、対象ファイルの内容やタイムスタンプが一切変更されないこと |
| **I3-2** | 依存先がない場合（空リスト） | 対象ファイルの `x-st-dependencies` が `[]`（空リスト） | `spectrack sync docs/a.md` | クラッシュせず、「同期する依存先がありません」等で安全に処理が完了すること |
| **I3-3** | バージョンダウングレードの追従 | `a.md` は `b.md:2.0.0` を参照しているが、`b.md` の Working tree が `1.5.0` に巻き戻っている | `spectrack sync docs/a.md` | 最新の Working tree 状態を正として、参照バージョンが `1.5.0` に同期（上書き）されること |

## 4. 異常系・ファイル不在・Git連携推定（Errors / Missing / Git Guessing）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E4-1** | 対象ファイルが未初期化 | `a.md` にフロントマターがない | `spectrack sync docs/a.md` | 「対象ファイルが初期化されていません」等のエラーとなり、ファイルが変更されないこと |
| **E4-2** | 依存先ファイルが完全に削除された（Git推定） | 依存先 `b.md` のファイルが削除されており、ID解決ができない。Git履歴には存在する | `spectrack sync docs/a.md` | 仕様書の定義通り `ERROR: 依存先 [id] が見つかりません。ファイルが削除された可能性があります。（Git履歴の推定元パス: docs/b.md）` が出力され、処理が中断されること |
| **E4-3** | 依存先ファイルがGit履歴にもない | `b.md` を作成・リンク直後に未コミットのまま削除し、ID解決ができない | `spectrack sync docs/a.md` | Git履歴からの推定に失敗するケース。「依存先 [id] が見つかりません。Git履歴からもパスを推定できませんでした」等のエラーで中断されること |

## 5. 不完全なフロントマター（Missing Keys / Invalid Formats）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **M5-1** | 対象ファイルの `x-st-dependencies` 欠損 | 対象ファイルに `x-st-id` はあるが、依存リストキー自体が存在しない | `spectrack sync docs/a.md` | Null参照等でクラッシュせず、「同期する依存先がありません」として安全に終了すること |
| **M5-2** | 依存先フロントマターの破壊（バージョン欠損） | 依存先 `b.md` のファイルは存在するが、バージョン情報（`version`キー等）が欠損している | `spectrack sync docs/a.md` | 依存先バージョンが取得できないため、「依存先 [b.md] のバージョン情報が見つかりません」等のエラーで対象ファイルの同期処理が中断されること |
| **M5-3** | 依存リスト内の不正な要素 | 対象ファイルの依存リストにオブジェクトではなく文字列が含まれている（例: `- st-b-001`） | `spectrack sync docs/a.md` | 対象ファイルのリスト形式不正を検知し、「依存リストの形式が不正です」等のエラーでファイルが変更されず安全に中断すること |

## 7. Dry-Run オプション（`--dry-run`）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **D7-1** | 通常の同期の Dry-Run | 依存先が更新されている | `spectrack sync docs/a.md --dry-run` | 標準出力に変更予定の差分（バージョンの上書き、パスの更新）が表示され、実際のファイルは変更されないこと |
| **D7-2** | `--only` との組み合わせ Dry-Run | 依存先 `b.md`, `c.md` が更新済み | `spectrack sync docs/a.md --only=docs/b.md --dry-run` | `b.md` のみが同期される予定の差分が出力され、`c.md` の差分は出力されないこと。実ファイルは変更されないこと |
