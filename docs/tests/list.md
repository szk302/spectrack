# `spectrack list` コマンド テストケース一覧

## 1. 基本動作（正常系：一覧表示）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **L1-1** | 追跡対象ファイルの一覧表示 | プロジェクト内に初期化・コミット済みの対象ファイルが複数存在する | `spectrack list` | 終了コード `0`。ファイルパス、ID、現在のバージョン、最終コミット日・ハッシュが全件表示されること |
| **L1-2** | 対象ファイル0件時の表示 | プロジェクト内に追跡対象ファイル（フロントマターを持つ対象拡張子）が1つも存在しない | `spectrack list` | 終了コード `0`。「追跡対象のドキュメントがありません」等のメッセージが表示され、正常終了すること |

## 2. Git連携とフォールバック（Graceful Degradation & Status）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **G2-1** | Git未初期化ディレクトリ | プロジェクトが `git init` されていない（`.git`ディレクトリがない） | `spectrack list` | 終了コード `0`。エラーにならず一覧が表示され、日時の部分が**OSの最終更新日時（mtime）**となり、`(Git未管理)` 等のステータスが併記されること |
| **G2-2** | Gitコミットゼロの状態 | `git init` 済みだが、まだ1つもコミットが存在しないリポジトリ | `spectrack list` | 終了コード `0`。上記と同様に、Git履歴がない前提でフォールバック表示（mtime等）され、正常終了すること |
| **G2-3** | 新規追加（Untracked）ファイルの混在 | 既存のコミット済みファイルに加え、`init` したばかりで一度もコミットされていないファイルがある | `spectrack list` | 終了コード `0`。Untrackedファイルはフォールバック表示（mtimeと `(新規)` 等）となり、他のファイルは正常にGit情報が表示されること |
| **G2-4** | 未コミットの変更検知（バージョン） | 既存ファイルのフロントマターの `version` を更新し、まだコミットしていない | `spectrack list` | 終了コード `0`。該当ファイルの表示において、バージョン表記の横等に `(未コミットの変更あり)` 等が明示されること |
| **G2-5** | 未コミットの変更検知（本文のみ） | バージョンは変わっていないが、ファイル本文のみを編集して保存している（Git上の差分あり） | `spectrack list` | 終了コード `0`。Working treeに変更があることを検知し、該当ファイルに `(未コミットの変更あり)` が明示されること |

| :--- | :--- | :--- | :--- | :--- |

## 4. 境界値・パス解決（Boundary & Paths）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **B4-1** | 非常に深いディレクトリ階層 | 対象ファイルが制限ギリギリの深い階層にある（例: `docs/a/b/c/.../z.md`） | `spectrack list` | 終了コード `0`。パス長制限や再帰処理の上限に引っかからず、正しくパスが解決されて一覧に表示されること |
| **B4-2** | 完全に空のファイル（0バイト） | 拡張子は `.md` だが、ファイルサイズが0バイトのファイルが存在する | `spectrack list` | ファイル読み込み時・パース時にクラッシュせず、未初期化（フロントマターなし）として安全にスキップされること |
| **B4-3** | 特殊文字・マルチバイト文字のパス | ファイル名やディレクトリ名にスペース、絵文字、日本語が含まれている | `spectrack list` | 文字化けやパス解決エラーを起こさず、ターミナル上で正しく一覧表示されること |

## 7. 異常系・ファイル破損（Errors / Invalid Formats）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E7-1** | 設定ファイル不在 | プロジェクトルートに `spectrack.yml` が存在しない | `spectrack list` | 終了コード `1`。「spectrack.yml が見つかりません。spectrack init を実行してください」の共通エラーが出力されること |
| **E7-2** | IDの重複（Duplicate IDs） | `a.md` と `b.md` で全く同じ `x-st-id` が使用されている | `spectrack list` | 終了コード `1`。スキャン時にIDの一意性制約違反を検知し、「ERROR: ID重複検出」を出力して処理が中断されること |
| **E7-3** | フロントマターの破損（パース失敗） | 拡張子が `.md` だが中身がバイナリ、またはYAMLとして完全に不正なファイルが存在する | `spectrack list` | ツール全体がクラッシュせず、エラーをキャッチして警告を出しつつ、**他の正常なファイルのリストは最後まで出し切ること** |
| **E7-4** | 必須キーの欠損（IDやバージョン） | `a.md` にフロントマターはあるが、`x-st-id` や `version` キーが欠損している | `spectrack list` | クラッシュせず、一覧内で該当項目の値が `[不明]` や `[未設定]` 等として安全に表示されるか、警告が出力されること |
