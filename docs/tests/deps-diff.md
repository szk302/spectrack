# `spectrack deps-diff` コマンド テストケース一覧

## 1. 基本動作（正常系：依存先差分のループ処理）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **DD1-1** | 単一の依存先の差分表示 | `a.md` は `b.md:1.0.0` に依存。現在の `b.md` は `1.1.0`（更新あり） | `spectrack deps-diff docs/a.md` | 終了コード `0`。`b.md` の `1.0.0` 時点と現在の Working Tree の差分が表示されること |
| **DD1-2** | 複数の依存先の連続表示 | `a.md` は `b.md:1.0.0`, `c.md:2.0.0` に依存し、両方とも現在更新されている | `spectrack deps-diff docs/a.md` | 終了コード `0`。`b.md` と `c.md` の両方の差分が、ヘッダ等で区切られて連続して表示されること |
| **DD1-3** | 更新がない依存先のスキップ | `a.md` は `b.md` (更新あり) と `c.md` (更新なし) に依存している | `spectrack deps-diff docs/a.md` | 終了コード `0`。更新がある `b.md` の差分のみが表示され、`c.md` はスキップされる（または「変更なし」と表示される）こと |
| **DD1-4** | 全依存先が更新なしの場合 | `a.md` のすべての依存先のバージョンが、現在の Working Tree と一致している | `spectrack deps-diff docs/a.md` | 終了コード `0`。「すべての依存先ドキュメントに変更はありません」等のメッセージが表示され正常終了すること |
| **DD1-5** | 依存先が0件の場合（空リスト） | `a.md` の `x-st-dependencies` が空配列 `[]` である | `spectrack deps-diff docs/a.md` | 終了コード `0`。「依存しているドキュメントはありません」等のメッセージで正常終了すること |

| :--- | :--- | :--- | :--- | :--- |

## 3. 表示オプションとHunk制御（Display Options & Hunks）

`diff` コマンドで検証したコアアルゴリズムが、複数ファイルに対しても正しく伝播するかをテストします。

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **O3-1** | `--context` オプションの伝播 | 依存先 `b.md` が離れた2箇所で更新されている | `spectrack deps-diff docs/a.md --context=1` | 終了コード `0`。指定したコンテキスト行数（1行）が依存先の差分計算に適用され、Hunk（変更ブロック）が正しく分割されて表示されること |
| **O3-2** | `--full` オプションの伝播 | 依存先 `b.md`, `c.md` が更新されている | `spectrack deps-diff docs/a.md --full` | 終了コード `0`。両方の依存先ファイルが「全体表示（フル出力）」の形式で連続して出力されること |

## 4. Git環境依存とエラーハンドリング（Git Dependency）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **G4-1** | Git未初期化ディレクトリでの実行 | プロジェクトが `git init` されていない | `spectrack deps-diff docs/a.md` | 終了コード `1`。「Git リポジトリが初期化されていません」の共通エラーを出力して安全に中断すること |
| **G4-2** | Gitコミットゼロの状態 | `git init` 済みだが、まだ1つもコミットが存在しない | `spectrack deps-diff docs/a.md` | 終了コード `1`。「少なくとも1つのコミットが必要です」のエラーを出力して中断されること |

## 5. 異常系・履歴探索と耐障害性（History Errors & Resilience）

複数ファイルをループ処理するため、「1つのファイルがおかしいからといって全体を道連れにしてクラッシュしないか」が重要です。

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E5-1** | 参照バージョンが履歴に存在しない | `a.md` が `b.md:99.9.9` に依存している（ユーザーがYAMLを手動で不正な値に書き換えた等） | `spectrack deps-diff docs/a.md` | **終了コード `1` または `2`**。`b.md` の履歴取得エラーを検知し、「b.md の指定バージョン (99.9.9) の履歴が見つかりません」等と警告またはエラーを出し、安全に処理をハンドリングすること |
| **E5-2** | 依存先ファイルが完全に削除されている | `a.md` が依存する `b.md` が削除され、Git履歴には存在するが Working Tree にはない | `spectrack deps-diff docs/a.md` | 終了コード `1`。`ERROR: 依存先 [id] が見つかりません。ファイルが削除された可能性があります` を出力し、Git履歴の推定元パスを提示して中断すること |
| **E5-3** | 依存リスト内の1ファイルがバイナリ化 | `a.md` は `b.md`, `c.md` に依存。`b.md` の中身がバイナリデータに置き換わっている | `spectrack deps-diff docs/a.md` | ツール全体がクラッシュ・文字化けせず、`b.md` は「バイナリのため表示不可」としてスキップまたは警告を出し、**正常な `c.md` の差分は出力しきること** |
| **E5-4** | 依存先のバージョン情報欠損 | 依存先 `b.md` にフロントマターはあるが、`version` キー自体が欠損している | `spectrack deps-diff docs/a.md` | クラッシュせず、「b.md のバージョン情報が取得できません」等の警告/エラーで安全にハンドリングすること |
| **E5-5** | 対象ファイル自体が未初期化 | `a.md` にフロントマターがない | `spectrack deps-diff docs/a.md` | 終了コード `1`。「対象ファイルが初期化されていません」で即座に中断されること |
