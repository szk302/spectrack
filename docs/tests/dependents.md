# `spectrack dependents` コマンド テストケース一覧

## 1. 基本動作（通常モード：Working Tree検索）

Git履歴を使用せず、現在のファイルシステム上にあるファイルのみをスキャンする正常系テストです。

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **D1-1** | 単一の依存元が存在 | 対象 `a.md` に依存しているファイルが `b.md` のみ存在する | `spectrack dependents docs/a.md` | 終了コード `0`。`b.md` が `a.md` に依存している旨が正常に表示されること |
| **D1-2** | 複数の依存元が存在 | 対象 `a.md` に依存しているファイルが複数存在する | `spectrack dependents docs/a.md` | 終了コード `0`。依存している全ファイルがリストアップされて表示されること |
| **D1-3** | 依存元が存在しない | 対象 `a.md` に依存しているファイルが現在のプロジェクト内に1つも存在しない | `spectrack dependents docs/a.md` | 終了コード `0`。「依存しているドキュメントはありません」等のメッセージが表示され正常終了すること |
| **D1-4** | 循環依存状態での検索 | `a.md` が `b.md` に依存し、`b.md` も `a.md` に依存している | `spectrack dependents docs/a.md` | 終了コード `0`。無限ループにならず、`b.md` が依存元として正しく1回だけ表示されること |
| **D1-5** | 未コミット変更の反映 | 依存元 `b.md` で `a.md` への依存を追加したが、未コミット状態である | `spectrack dependents docs/a.md` | 終了コード `0`。Working Treeを正とするため、未コミットであっても即座に `b.md` が依存元としてヒットすること |

## 2. `--all` オプション動作（歴史的依存関係の検索）

Gitの全コミット履歴（`git log -S` 等）をスキャンし、過去の依存状態を復元する正常系テストです。

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **H2-1** | 削除された依存元ファイルの特定 | 過去に `a.md` に依存していた `old.md` が存在したが、現在はファイルごと削除されている | `spectrack dependents docs/a.md --all` | 終了コード `0`。現在の依存元に加え、`old.md` が「過去に依存（削除済み）」等のステータスで表示されること |
| **H2-2** | 依存が解除されたファイルの特定 | `c.md` は過去に `a.md` に依存していたが、現在のWorking Treeでは依存リストから削除されている | `spectrack dependents docs/a.md --all` | 終了コード `0`。`c.md` が「過去に依存（依存解除）」等のステータスで、解除されたコミット情報と共に表示されること |
| **H2-3** | 履歴と現在の重複排除 | `b.md` は過去から現在まで一貫して `a.md` に依存し続けている | `spectrack dependents docs/a.md --all` | 終了コード `0`。`b.md` が「現在も依存中」として1件のみ表示され、過去の全コミット分が重複表示されたりしないこと |

## 3. Git環境依存とフォールバック（Environment & Graceful Degradation）

コマンドのモードによってGitエラーを出すか、フォールバックさせるかの厳密な切り分けテストです。

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **G3-1** | 【通常】Git未初期化ディレクトリ | プロジェクトが `git init` されていない | `spectrack dependents docs/a.md` | **終了コード `0`**。Gitに依存しないため、エラーにならず現在のファイルシステムから正常に逆引き結果が表示されること |
| **G3-2** | 【--all】Git未初期化ディレクトリ | プロジェクトが `git init` されていない | `spectrack dependents docs/a.md --all` | **終了コード `1`**。全履歴検索にはGitが必須なため、「Git リポジトリが初期化されていません」の共通エラーを出力して中断されること |
| **G3-3** | 【--all】Gitコミットゼロの状態 | `git init` 済みだが、まだ1つもコミットが存在しない | `spectrack dependents docs/a.md --all` | **終了コード `1`**。仕様書の定義通り「少なくとも1つのコミットが必要です」のエラーを出力して中断されること |

## 4. 境界値・パス解決・除外設定（Boundary / Paths / Filtering）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **B4-1** | 相対・絶対パスの解決 | 対象ファイルを `../docs/a.md` や絶対パスで指定する | `spectrack dependents ../docs/a.md` | 指定方法に関わらず内部の「一意なID」に正しく解決され、漏れなく依存元が検索されること |
| **B4-2** | サブディレクトリからの実行 | カレントディレクトリを `docs/prd/` に移動している | `spectrack dependents requirements.md` | 親ディレクトリを遡って `spectrack.yml` を発見し、プロジェクト全体のファイルを正しくスキャンできること |
| **B4-3** | 除外設定（`.spectrackignore`） | `a.md` に依存している `b.md` が除外ディレクトリ配下にある | `spectrack dependents docs/a.md` | 除外リスト配下のファイルはスキャン対象外となるため、`b.md` は依存元として表示されないこと（`--all` 指定時の履歴スキャンで除外設定をどう扱うかは仕様によるが、通常モードでは除外されること） |

## 5. 異常系・耐障害性（Errors / Resilience）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E5-1** | 対象ファイルが存在しない | 存在しないファイルパスを指定する | `spectrack dependents docs/not_found.md` | 終了コード `1`。「ファイルが見つかりません」のエラーが出力されること |
| **E5-2** | 対象ファイルが未初期化 | `a.md` にフロントマターがない | `spectrack dependents docs/a.md` | 終了コード `1`。「対象ファイルが初期化されていません（IDが取得できません）」等のエラーとなり中断されること |
| **E5-3** | 他ファイルの破損による巻き込み防止 | 対象 `a.md` と依存元 `b.md` は正常だが、全く関係ない `broken.md` の中身がバイナリ等で破損している | `spectrack dependents docs/a.md` | **終了コード `0` または `2`**。全体スキャン中に無関係なファイルのパースに失敗してもツール全体がクラッシュせず、警告を出しつつ `b.md` の検索結果を正しく出力しきること |
| **E5-4** | 依存先リストの型不正（スキャン耐性） | スキャン対象の別ファイル `x.md` の `x-st-dependencies` が文字列やオブジェクトになっている | `spectrack dependents docs/a.md` | スキャン中に配列メソッド（`.includes`等）の呼び出しで `TypeError` を起こさず、型不正を安全に無視・スキップすること |
