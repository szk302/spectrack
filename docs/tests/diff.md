# `spectrack diff` コマンド テストケース一覧

## 1. 基本動作（正常系：バージョン間差分の表示）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **D1-1** | 直前バージョンとの比較（デフォルト） | 対象 `a.md` の過去の最新バージョンが `1.0.0`。現在は本文が変更され、バージョンが `1.1.0` | `spectrack diff docs/a.md` | 終了コード `0`。`1.0.0` 時点のファイル状態と現在の差分が、標準のGit diff形式（前後3行のコンテキスト）で表示されること |
| **D1-2** | 指定バージョンとの比較 | `a.md` の履歴に `1.0.0`, `1.1.0` があり、現在が `2.0.0` | `spectrack diff docs/a.md --version=1.0.0` | 終了コード `0`。指定した `1.0.0` 時点のコミットが特定され、現在との差分が正しく表示されること |
| **D1-3** | 変更がない場合の表示 | 現在のWorking Treeが、前回のバージョン（コミット）から一切変更されていない | `spectrack diff docs/a.md` | 終了コード `0`。「変更はありません」等のメッセージが表示され、正常終了すること |
| **D1-4** | 未コミットの変更のみ存在する場合 | バージョン番号は `1.0.0` のままだが、本文（Working Tree）のみが変更されている | `spectrack diff docs/a.md` | 終了コード `0`。最後に `1.0.0` になったコミットと、現在の未コミットの本文との差分が正しく表示されること |

## 2. 複数箇所の変更とHunk制御（Multiple Hunks & Context Merging）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **M2-1** | 離れた複数箇所の変更（Hunkの分割） | 対象 `a.md` の10行目と50行目の2箇所が変更されている | `spectrack diff docs/a.md` | 終了コード `0`。中間の変更されていない行が正しく省略され、`@@` で始まる2つの独立した変更ブロックとして表示されること |
| **M2-2** | 隣接・近接する複数箇所の変更（Hunkの結合） | 対象 `a.md` の10行目と12行目が変更されている（デフォルトのcontext=3の範囲内） | `spectrack diff docs/a.md` | 終了コード `0`。コンテキスト行が重複出力されず、10行目〜12行目を包含する1つの変更ブロックに結合されて表示されること |
| **M2-3** | `--context` オプションによるHunk結合の変化 | 10行目と15行目が変更されている（間隔が4行） | `spectrack diff docs/a.md --context=1`<br>および `--context=5` | 終了コード `0`。`--context=1` では2つの独立したブロックに分かれ、`--context=5` では間の行も含まれ1つのブロックに結合されること |
| **M2-4** | ファイル先頭と末尾の変更 | `a.md` の1行目（フロントマター直下等）と、ファイルの最終行が同時に変更されている | `spectrack diff docs/a.md` | 終了コード `0`。ファイル境界（BOF/EOF）におけるコンテキスト行の計算で配列外参照エラー等を起こさず、正しく2つのブロックが表示されること |

## 3. 表示オプションの制御（Display Options）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **O3-1** | `--context` の指定（行数制御） | `a.md` が大きく変更されている | `spectrack diff docs/a.md --context=1` | 終了コード `0`。変更箇所の前後1行のみをコンテキストとして差分がコンパクトに表示されること |
| **O3-2** | `--context=0` の指定 | `a.md` が変更されている | `spectrack diff docs/a.md --context=0` | 終了コード `0`。コンテキスト行を含まず、変更された行（`+` / `-`）のみが表示されること |
| **O3-3** | `--full` の指定（全体表示） | `a.md` が変更されている | `spectrack diff docs/a.md --full` | 終了コード `0`。ファイル全体が出力され、変更箇所がインライン（`+`/`-`）で表示されること |
| **O3-4** | オプションの競合・不正値 | `a.md` が存在する | `spectrack diff docs/a.md --full --context=5` | 終了コード `1`。排他オプションの同時指定エラーになる、あるいは `--full` が優先される等、仕様通りの一貫した振る舞いをすること |

## 4. Git環境依存とエラーハンドリング（Git Dependency）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **G4-1** | Git未初期化ディレクトリでの実行 | プロジェクトが `git init` されていない | `spectrack diff docs/a.md` | 終了コード `1`。生のGitエラーではなく、「ERROR: Git リポジトリが初期化されていません」の共通エラーを出力して安全に中断すること |
| **G4-2** | Gitコミットゼロの状態 | `git init` 済みだが、まだ1つもコミットが存在しない | `spectrack diff docs/a.md` | 終了コード `1`。「ERROR: spectrack は Git の履歴を利用するため、少なくとも1つのコミットが必要です」を出力して中断されること |

## 5. 境界値・履歴探索の異常系（Boundary / History Errors）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **H5-1** | 存在しない過去バージョンの指定 | `a.md` の履歴に `5.0.0` は存在しない | `spectrack diff docs/a.md --version=5.0.0` | 終了コード `1`。「指定されたバージョン (5.0.0) の履歴が見つかりません」等のエラーを出力して中断されること |
| **H5-2** | 履歴が1つしかない（初回コミット） | `a.md` が `1.0.0` としてコミットされた直後であり、過去のバージョンが存在しない | `spectrack diff docs/a.md` | 終了コード `1` または `0`。「比較対象となる過去のバージョンが存在しません」という旨のメッセージで安全に処理を終了すること |
| **H5-3** | フロントマターのみの変更 | 本文は全く同じだが、フロントマターの `version` や依存リストだけが更新されている | `spectrack diff docs/a.md` | 終了コード `0`。フロントマター部分のみの差分が正確に検知・表示されること |
