# `spectrack status` コマンド テストケース一覧

## 1. 基本動作（正常系・更新なし / Exit Code: 0）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **S1-1** | 単一ファイルの依存状況表示 | 対象ファイル `a.md` の依存先 `b.md` がすべて最新（参照バージョンとWorking treeが同一） | `spectrack status docs/a.md` | 終了コード `0`。ツリー上に `✅` とともに現在のバージョンが表示されること |
| **S1-2** | プロジェクト全体の依存状況表示 | 引数なし。プロジェクト内の全ファイルの依存先がすべて最新 | `spectrack status` | 終了コード `0`。プロジェクト内の全対象ファイルの依存ツリーが表示され、すべて正常であること |
| **S1-3** | 依存先が存在しないファイルの表示 | `a.md` にフロントマターはあるが、依存先が登録されていない（空リスト） | `spectrack status docs/a.md` | 終了コード `0`。依存先がない旨（または自身のみ）が表示され、正常終了すること |

## 2. 更新検出（警告 / Exit Code: 2）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **U2-1** | メジャーバージョンの更新検知 | 依存先 `b.md` が `1.0.0` から `2.0.0` に更新されている（未コミット） | `spectrack status docs/a.md` | 終了コード `2`。ツリー上に `⚠️ 更新あり` の警告が表示されること |
| **U2-2** | マイナーバージョンの更新検知 | 依存先 `b.md` が `1.0.0` から `1.1.0` に更新されている（未コミット） | `spectrack status docs/a.md` | 終了コード `2`。ツリー上に `⚠️ 更新あり` の警告が表示されること |
| **U2-3** | `0.x.x` 帯のマイナー更新検知 | 依存先 `b.md` が `0.1.0` から `0.2.0` に更新されている | `spectrack status docs/a.md` | 終了コード `2`。仕様通り `0.x.x` のマイナー更新は「破壊的変更」として扱われ、更新ありの警告が出ること |
| **U2-4** | 複数依存先の混在状態 | 依存先 `b.md` は最新、`c.md` は更新済み | `spectrack status docs/a.md` | 終了コード `2`。`b.md` には `✅`、`c.md` には `🔄/⚠️` が正しく出し分けられて表示されること |

## 3. SemVerルールと `--strict` オプション（Boundary / Rules）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **V3-1** | パッチ更新の無視（デフォルト挙動） | 依存先 `b.md` が `1.0.0` から `1.0.1` に更新されている | `spectrack status docs/a.md` | 終了コード `0`。仕様に基づき、パッチ更新は更新とみなされず `✅` として表示されること |
| **V3-2** | `--strict` 指定時のパッチ更新検知 | 依存先 `b.md` が `1.0.0` から `1.0.1` に更新されている | `spectrack status docs/a.md --strict` | 終了コード `2`。`--strict` 指定によりパッチ更新も厳密に検知され、`⚠️ 更新あり` となること |
| **V3-3** | 無効なSemVerの検知 | 依存先 `b.md` のWorking treeバージョンが `1.0` などの不正な形式 | `spectrack status docs/a.md` | 終了コード `2`。仕様書定義通り「WARNING: [file] のバージョン [version] は有効なセマンティックバージョンではありません」が出力されること |

## 4. 異常系・Git連携・ファイル不在（Errors / Exit Code: 1）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E4-1** | 対象ファイルが存在しない | 指定したファイルが存在しない | `spectrack status docs/not_found.md` | 終了コード `1`。「ファイルが見つかりません」のエラーが出力されること |
| **E4-2** | 対象ファイルが未初期化 | `a.md` にフロントマターがない | `spectrack status docs/a.md` | 終了コード `1`。「対象ファイルが初期化されていません」等のエラーとなること |
| **E4-3** | 依存先ファイルが削除されている（Git推定） | 依存先 `b.md` のファイルが削除されており、ID解決ができない。Git履歴には存在する | `spectrack status docs/a.md` | 終了コード `1`。`ERROR: 依存先 [id] が見つかりません。ファイルが削除された可能性があります。（Git履歴の推定元パス: ...）` が出力されること |
| **E4-4** | Git未初期化のディレクトリでの実行 | プロジェクトがGitリポジトリとして `git init` されていない | `spectrack status` | 終了コード `1`。仕様書定義通り「ERROR: Git リポジトリが初期化されていません」が出力されること |

## 5. 循環依存の取り扱い（Cyclic Dependencies）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **C5-1** | 循環依存状態での正常な出力 | `a.md` が `b.md` に依存し、`b.md` が `a.md` に依存。どちらも更新なし | `spectrack status docs/a.md` | 終了コード `0`。無限ループ（スタックオーバーフロー）に陥ることなく、ツリーが適切に省略または表示され正常終了すること |
| **C5-2** | 循環依存状態での更新検知 | 上記の状態で、`b.md` が更新されている | `spectrack status docs/a.md` | 終了コード `2`。循環依存下であっても、`b.md` の更新を正しく検知し警告を出力すること |

## 6. 不完全なフロントマター・ファイル種別不整合（Formats / Invalid Types）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **F6-1** | 依存先のバージョン情報欠損 | 依存先 `b.md` は存在するが、バージョンキーが欠損している | `spectrack status docs/a.md` | 終了コード `1`。「依存先のバージョン情報が見つかりません」としてエラー終了すること |
| **F6-2** | 依存元のリストキー自体が欠損 | 対象ファイルの `x-st-dependencies` が存在しない | `spectrack status docs/a.md` | クラッシュせず、空リストとして扱われ、終了コード `0` で正常終了すること |
| **F6-3** | バイナリ偽装ファイル | 対象ファイル、または依存先ファイルがバイナリ | `spectrack status fake.md` | 終了コード `1`。ASTパース等のエラーをキャッチし、クラッシュせず安全にエラーを出力すること |
| **F6-4** | IDキー欠損（空文字） | 依存先 `b.md` のフロントマターに `x-st-id: ""` が設定されている | `spectrack status docs/a.md` | 終了コード `1`。IDによる解決が不可能なため、「IDが不正または未初期化です」等でエラーになること |
