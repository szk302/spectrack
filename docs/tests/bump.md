# `spectrack bump` コマンド テストケース一覧

## 1. 基本動作（正常系：SemVer更新）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **B1-1** | メジャーバージョンの更新 | 対象ファイルのバージョンが `1.2.3` | `spectrack bump docs/a.md --major` | バージョンが `2.0.0` に更新されること |
| **B1-2** | マイナーバージョンの更新 | 対象ファイルのバージョンが `1.2.3` | `spectrack bump docs/a.md --minor` | バージョンが `1.3.0` に更新されること |
| **B1-3** | パッチバージョンの更新 | 対象ファイルのバージョンが `1.2.3` | `spectrack bump docs/a.md --patch` | バージョンが `1.2.4` に更新されること |
| **B1-4** | `0.x.x` の更新（初期開発フェーズ） | 対象ファイルのバージョンが `0.1.2` | `spectrack bump docs/a.md --minor` | バージョンが `0.2.0` に更新されること（`0.x.x`帯でもSemVerのルール通りに計算されること） |
| **B1-5** | AST（フォーマット・コメント）の維持 | バージョン表記の横にYAMLコメントがある（例: `version: 1.0.0 # 初期版`） | `spectrack bump docs/a.md --minor` | 値のみが `1.1.0` に書き換わり、コメントやインデント形式がそのまま維持されること |

## 3. バージョン仕様の境界値・特殊ルール（SemVer Boundaries / Rules）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **S3-1** | 許容されるプレリリース表記の更新 | 対象ファイルのバージョンが `1.0.0-1`（数値のみのプレリリース） | `spectrack bump docs/a.md --patch` | SemVerの仕様に従い `1.0.0` （または内部仕様に基づく適切な次のバージョン）に更新され、エラーにならないこと |
| **S3-2** | 不正なプレリリース表記の検知 | 対象ファイルのバージョンが `1.0.0-alpha`（英字を含む） | `spectrack bump docs/a.md --minor` | 仕様に基づくエラー「プレリリースバージョンは数値のみ許可されています」が出力され、変更が中断されること |
| **S3-3** | 既存バージョンがSemVer非準拠 | 対象ファイルのバージョンが `1.0`（パッチ不足）や `v1.0.0`（vプレフィックス付き）や `a.b.c` | `spectrack bump docs/a.md --major` | 既存バージョンのパースに失敗し、「現在のバージョンが有効なセマンティックバージョンではありません」等のエラーで中断されること |

## 4. ネストされたバージョンパスの解決（Nested Version Paths）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **P4-1** | ネストされたキー（`info.version`）の更新 | 対象が `.yml` で `x-st-version-path: info.version` と設定されている | `spectrack bump docs/api.yml --minor` | YAML内の `info` 階層下にある `version` キーの値だけが正確に特定・更新されること |
| **P4-2** | 存在しないパス階層の指定 | `x-st-version-path: metadata.docs.version` だが、実際のYAMLに `metadata` キーが存在しない | `spectrack bump docs/a.md --patch` | プロパティ未定義（undefined）でツールがクラッシュせず、「指定されたバージョンパスが見つかりません」として安全にエラー終了すること |

## 5. 異常系・ファイル種別不整合（Errors / Invalid Formats）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **E5-1** | 対象ファイルが未初期化 | `a.md` にフロントマターがない | `spectrack bump docs/a.md --minor` | 「対象ファイルが初期化されていません」等のエラーとなり、ファイルが変更されないこと |
| **E5-2** | 存在しないファイルの指定 | 指定したファイルが存在しない | `spectrack bump docs/not_found.md --minor` | 「ファイルが見つかりません」のエラーとなり、処理が中断されること |
| **E5-3** | 対象外拡張子の指定 | `logo.png` が存在する | `spectrack bump logo.png --major` | サポート外の拡張子である旨のエラーとなり、処理が中断されること |
| **E5-4** | バイナリ偽装ファイル | 拡張子は `.md` だが中身がバイナリ | `spectrack bump fake.md --minor` | ASTパース時にエラーを安全に検知し、ファイルが破壊されずに終了すること |

## 6. 不完全なフロントマター（Missing Keys / Empty Values）

| ID | テスト項目 | 事前条件 | 実行コマンド | 期待される結果 |
| :--- | :--- | :--- | :--- | :--- |
| **M6-1** | `x-st-version-path` キー自体の欠損 | フロントマターに `x-st-id` はあるが、`x-st-version-path` キーが存在しない | `spectrack bump docs/a.md --minor` | どこを参照すべきか不明なため、「バージョンパスが設定されていません」等のエラーで中断されること |
| **M6-2** | `x-st-version-path` の値が空文字 | `x-st-version-path: ""` と設定されている | `spectrack bump docs/a.md --patch` | 不正なパス指定として検知され、エラーで中断されること |
| **M6-3** | バージョン値の欠損（空文字・null） | パスは正しいが、実際の値が `version: ""` または `version: null` となっている | `spectrack bump docs/a.md --major` | バージョン情報のパースに失敗し、「現在のバージョンが取得できません（空です）」等のエラーで安全に中断されること |
| **M6-4** | バージョン値の型不正 | バージョンの値が配列 `version: [1,0,0]` やオブジェクトになっている | `spectrack bump docs/a.md --minor` | 文字列以外の不正な型であることを検知し、クラッシュせずにエラーで処理を中断すること |
